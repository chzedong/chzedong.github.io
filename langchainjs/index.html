LangChain.js — LLM 应用框架

TypeScript 框架，通过标准接口串联模型、工具、向量存储等组件，简化 AI 应用开发。

v1.4.4 MIT License TypeScript pnpm Monorepo LangChain 33+ Providers

1. 项目背景

LangChain.js 是 LangChain AI 开源的 JavaScript/TypeScript 框架，目标是成为"构建 LLM 应用的平台"。它提供一套统一的抽象接口——Runnable、Message、Tool、VectorStore 等——让开发者用一致的 API 对接 33+ 个模型提供商和工具集成。

🎯 设计哲学

LangChain.js 的核心不是一个具体功能，而是一套可组合的接口规范。任何实现 Runnable 接口的组件都可以通过 .pipe()、.bind()、.withFallbacks() 等方法自由编排——这就是 LangChain 的核心竞争力：所有组件共享同一套编排原语。

核心价值：

  • 模型互操作性 — 统一接口抹平 OpenAI / Anthropic / Google 等提供商的差异
  • 实时数据增强 — RAG（检索增强生成）原生支持，向量存储、文档加载器一站式
  • Agent 编排 — 工具调用、多步推理、Human-in-the-loop 的完整基础设施
  • 跨环境运行 — Node.js / Browser / Cloudflare Workers / Deno / Bun 全覆盖

2. 快速上手

基础调用 — 3 行代码调用 OpenAI
import { ChatOpenAI } from "@langchain/openai"

const model = new ChatOpenAI({ model: "gpt-4o" })
const response = await model.invoke("Hello, world!")
Chain — 管道编排多个步骤
import { ChatPromptTemplate } from "@langchain/core/prompts"

const prompt = ChatPromptTemplate.fromTemplate(
  "Tell me a joke about {topic}"
)
const chain = prompt.pipe(model)    // ★ Runnable 管道

const result = await chain.invoke({ topic: "programmers" })
Tool Calling — 模型调用外部工具
import { tool } from "@langchain/core/tools"
import { z } from "zod/v3"

const weatherTool = tool(
  async ({ city }) => `The weather in ${city} is sunny`,
  { name: "get_weather", schema: z.object({ city: z.string() }) }
)

const agent = model.bindTools([weatherTool])
const response = await agent.invoke("What's the weather in Tokyo?")
// → response.tool_calls[0].name === "get_weather"

5. 核心链路

5.1 Runnable — 一切皆可编排

LangChain.js 的核心思想：所有组件都实现 Runnable 接口。无论是 LLM 调用、Prompt 模板、还是 Agent 决策——它们共享相同的 invoke / stream / batch 方法。

5.2 Agent 执行流程

sequenceDiagram
    participant U as User
    participant A as Agent
    participant M as ChatModel
    participant T as Tools
    participant MW as Middleware

    U->>A: invoke("query")
    A->>MW: beforeModel hook
    MW->>A: modified messages
    loop Agent Loop
        A->>M: invoke(messages)
        M-->>A: response (text / tool_calls)
        A->>MW: afterModel hook
        alt has tool_calls
            A->>T: invoke(tool_call)
            T-->>A: ToolMessage
            A->>MW: afterTool hook
        else no tool_calls
            A-->>U: final response
        end
    end

LangChain.js 的精髓在于 Runnable 接口——它用一个抽象统一了模型调用、工具执行、链式编排和 Agent 决策，使得整个生态系统可以像搭积木一样自由组合。
