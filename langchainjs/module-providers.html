🔌 Provider 集成模式

33+ Provider 的架构模式 — Chat Model 实现、向量存储对接、工具集成。

生态33+ Provider多模态

1. 模块定位

LangChain.js 的 Provider 生态是其最大优势之一。33 个 provider 包涵盖了聊天模型、嵌入模型、向量存储、工具四大类别。所有 Provider 通过实现相同的基类接口来保证互操作性。

2. 集成模式

每个 Provider 包遵循统一的结构：

libs/providers/langchain-{name}/
├── package.json        // export map: chat_models / embeddings / vectorstores / ...
├── src/
│   ├── chat_models.ts  // extends BaseChatModel
│   ├── embeddings.ts   // extends Embeddings
│   ├── vectorstores.ts  // extends VectorStore (如适用)
│   ├── tests/
│   │   ├── {name}.test.ts        // 单元测试
│   │   └── {name}.int.test.ts    // 集成测试（需 API key）
│   └── index.ts
└── tsdown.config.ts

3. Chat Model 实现

所有聊天模型 Provider 都继承自 BaseChatModel，只需实现少数抽象方法：

  • _generate(messages, options, runManager) — 核心调用逻辑
  • _streamResponseChunks(messages, options, runManager) — 流式输出
  • Provider 特有的参数（temperature、top_p 等）作为构造函数参数传入
💡 关键设计

LangChain 使用 lc_serializable / lc_namespace 等命名空间属性实现可序列化性——任何 ChatModel 实例都可以被序列化为 JSON 并在另一个运行时中反序列化恢复。这是 LangSmith 追踪和 LangServe 部署的基础。

4. 向量存储对接

向量存储 Provider（Pinecone、Qdrant、Weaviate、MongoDB Atlas、Redis、Neo4j 等）实现 VectorStore 抽象：

  • addDocuments(docs) — 存储文档向量
  • similaritySearch(query, k) — 相似性搜索
  • asRetriever() — 转换为 Runnable Retriever
