🤖 Agent & Middleware

langchain 主包 — Agent 执行引擎、中间件系统、Human-in-the-loop、LangChain Hub。

主包v1.4.4Agent-first

1. 模块定位

如果说 @langchain/core 提供了积木（Runnable、Message、Tool），那么 langchain 主包就是把积木搭成房子的说明书。它提供了开箱即用的 Agent、Middleware 系统和 LangChain Hub 集成。

2. Agent 执行引擎

Agent 是一个封装了"模型 → 工具调用 → 观察结果 → 继续推理"循环的 Runnable。核心流程：

flowchart TD
    A["创建 Agent"] --> B["调用 model.invoke(messages)"]
    B --> C{有 tool_calls?}
    C -->|是| D["执行工具"]
    D --> E["追加 ToolMessage"]
    E --> B
    C -->|否| F["返回最终 AIMessage"]

图：Agent ReAct 循环 — 模型推理与工具执行交替进行

3. Middleware 中间件系统

Middleware 是 LangChain.js Agent 的洋葱模型钩子系统——每个中间件可以在 Agent 生命周期的关键节点插入逻辑。中间件通过 hook 方法与 Agent 交互：

中间件文件功能
contextEditingcontextEditing.ts在模型调用前修改消息上下文（裁剪/注入）
dynamicSystemPromptdynamicSystemPrompt.ts根据运行时状态动态生成系统提示
hitlhitl.tsHuman-in-the-loop：工具调用前请求人工审批
llmToolSelectorllmToolSelector.ts用 LLM 筛选相关工具（减少 token 消耗）
modelCallLimitmodelCallLimit.ts限制最大模型调用次数
modelFallbackmodelFallback.ts模型调用失败时切换到备用模型
modelRetrymodelRetry.ts自动重试失败调用（指数退避）
🧠 Middleware 设计的巧妙之处

Middleware 不是在 Agent 外部包装——它们是注入到 Agent 内部的钩子。每个中间件实现 beforeModel / afterModel / beforeTool / afterTool 等 hook，Agent 在执行循环的对应阶段依次调用它们。这使得 Agent 的行为可以被第三方库扩展，而无需修改 Agent 自身源码。

4. LangChain Hub

LangChain Hub 是社区共享的 Prompt 仓库。通过 langchain/hub 导出的 pull() 函数，可以一行代码拉取社区发布的 Prompt 模板：

import { pull } from "langchain/hub"
const prompt = await pull("hwchase17/react")

5. UniversalChatModel

langchain/chat_models/universal 提供了一种统一模型接口——开发者可以通过一个标准 API 调用任意 Provider 的模型，而不需要分别导入各个 Provider 包。底层通过动态加载 + Provider 注册机制实现。

6. "如果我来写" 对比思考

🧠 问题：如何让 Agent 的行为可扩展？

直觉方案：在 Agent 类中提供大量配置选项（布尔标志）。

LangChain 的做法：Middleware 系统。Agent 在执行循环中依次调用所有注册中间件的 hook——这本质上是责任链模式。

差异分析：配置标志会爆炸（modelCallLimit、modelRetry、contextEditing、hitl...每个都需要一个 flag）。Middleware 让每个行为变化成为一个独立的类，Agent 本身保持简洁，扩展性由组合提供。
